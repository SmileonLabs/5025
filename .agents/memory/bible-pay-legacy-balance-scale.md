# 레거시 잔액·보상 스케일 불일치

bible-pay는 충전 시 결제 원금에 `POINTS_PER_KRW`(=10)을 곱해 포인트를 적립한다(1,000원 결제 → 10,000P). 이 전환 **이전에** 적립된 모든 값은 원화 스케일이라, 코드 변경만으로는 자동 재평가되지 않는다.

**영향 대상(원화 스케일로 남아 포인트로 재해석되면 10배 평가절하됨):**
- `parents.balance`, `children.balance`
- 미션 보상값(mission rewards)
- 전환 시점에 이미 존재하던 `gifticon_orders` 행의 `price`/`faceValue` 스냅샷

**Why:** 단위 라벨만 '원'→'P'로 바꿨을 뿐, 기존 숫자는 그대로다. 새 충전만 ×10이 적용되므로 신규/레거시 데이터가 같은 화면에서 10배 차이로 섞인다.

**How to apply:** 레거시 데이터를 살려야 한다면 일회성 `UPDATE ... SET balance = balance * 10`(parents·children·미션 보상)을 명시적으로 실행할 것.

**결정(2026-06-10):** 사용자가 기존 숫자를 **그대로 두기**로 확정 — ×10 마이그레이션도, 초기화도 하지 않음. 즉 dev DB의 기존 잔액/보상은 원화 시절 숫자 그대로 포인트로 해석된다(신규 충전만 ×10). 다시 묻지 말 것. 운영 데이터가 새로 생기는 시점부터는 ×10이 일관 적용되므로 추가 조치 불필요.
